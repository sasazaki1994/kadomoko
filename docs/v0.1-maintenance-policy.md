# KadoMoco v0.1 maintenance and feature freeze

From publication of v0.1.0 through v0.1.1, the release line is feature-frozen. Changes must be narrowly scoped, preserve `package.json` versioning rules and save version 9 unless a separately reviewed compatibility fix requires otherwise, and retain compatibility with existing saves and the production sprite pipeline.

## Allowed changes

- Crash, security, or serious accessibility fixes
- Save corruption or migration fixes
- Transparent rendering or window-position fixes
- Tray and shutdown-path fixes
- Installation, update, uninstall, or save-retention fixes
- CPU or memory regressions
- Corrections to typos or misleading documentation

## Deferred changes

Do not merge new care features, stats, save structures, level-cap extensions, companions, homes or props, sound, long-term progression, major UI changes, or broad balance changes into v0.1.1. Track such requests in a separate issue or future v0.2 milestone.

## Qualification rule

Every maintenance change requires focused regression coverage and the applicable release gates. A save-related change must demonstrate existing-save compatibility. A Windows lifecycle, packaging, rendering, or performance change must be retested on the affected real Windows versions; CI is not a substitute for real-device evidence.
